# <a name="call-resource-type"></a><span data-ttu-id="0f948-103">Tipo de recurso de chamada</span><span class="sxs-lookup"><span data-stu-id="0f948-103">call resource type</span></span>

[!INCLUDE [beta-disclaimer](../../includes/beta-disclaimer.md)]

<span data-ttu-id="0f948-104">O recurso **call** é criado quando há uma chamada recebida para o aplicativo ou quando o aplicativo cria uma nova chamada realizada por meio de um `POST` em `app/calls`.</span><span class="sxs-lookup"><span data-stu-id="0f948-104">The **call** resource is created when there is an incoming call for the application or the application creates a new outgoing call via a `POST` on `app/calls`.</span></span>

<span data-ttu-id="0f948-105">As chamadas podem ser configuradas como uma chamada ponto a ponto ou com vários participantes.</span><span class="sxs-lookup"><span data-stu-id="0f948-105">Calls can be set up as a peer-to-peer or as a multiparty call.</span></span> <span data-ttu-id="0f948-106">Para criar ou ingressar em uma chamada com vários participantes, forneça `chatInfo` e `meetingInfo`.</span><span class="sxs-lookup"><span data-stu-id="0f948-106">For creating or joining a multiparty call, supply the `chatInfo` and `meetingInfo`.</span></span> <span data-ttu-id="0f948-107">Caso não sejam fornecidos, uma nova reunião ad hoc é criada automaticamente.</span><span class="sxs-lookup"><span data-stu-id="0f948-107">If these are not supplied, a new ad hoc meeting is created automatically.</span></span> <span data-ttu-id="0f948-108">Para uma chamada recebida, grave esses valores em um armazenamento altamente disponível para que o aplicativo reingresse na chamada caso enfrente uma falha.</span><span class="sxs-lookup"><span data-stu-id="0f948-108">For an incoming call, record these values in a highly available store, so that your application to rejoin the call in the event your application crashes.</span></span>

<span data-ttu-id="0f948-109">Embora a mesma identidade não possa ser convidada várias vezes, é possível que um aplicativo ingresse na mesma reunião várias vezes.</span><span class="sxs-lookup"><span data-stu-id="0f948-109">Although the same identity cannot be invited multiple times, it is possible for an application to join the same meeting multiple times.</span></span> <span data-ttu-id="0f948-110">Toda vez que o aplicativo ingressa, uma chamada distinta `id` é fornecida para essa chamada à reunião.</span><span class="sxs-lookup"><span data-stu-id="0f948-110">Each time the application joins, a distinct call `id` is provided for that call to the meeting.</span></span> <span data-ttu-id="0f948-111">É recomendável usar identidades separadas para ingressar na reunião para que os clientes as exibam como participantes diferentes.</span><span class="sxs-lookup"><span data-stu-id="0f948-111">We recommend that you use separate identities to join the meeting in order for the clients to display them as different participants.</span></span>
